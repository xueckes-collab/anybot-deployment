# 聊天机器人上线部署指南

将聊天机器人接入已上线的官网，需要完成两步：**1. 部署机器人后端** 和 **2. 在官网加入脚本**。

---

## 一、部署机器人后端（API 服务）

机器人后端需要部署到一台可从公网访问的服务器，并得到一个 URL（如 `https://chat.anywayflooring.com`）。

### 方案 A：Railway（推荐，简单）

1. 注册 [Railway](https://railway.app)
2. 新建项目，选择「Deploy from GitHub」或上传 `chatbot` 文件夹
3. 配置启动命令：`cd backend && uvicorn main:app --host 0.0.0.0 --port $PORT`
4. 在 Railway 后台设置环境变量：
   - `OPENAI_API_KEY`（必填）
   - `OPENAI_BASE_URL`（若用国内中转，填写 API 地址）
   - `OPENAI_MODEL`（默认 `deepseek-chat` 或 `gpt-4o`）
5. 部署完成后，记下生成的域名，例如：`https://xxx.railway.app`

### 方案 B：Render

1. 注册 [Render](https://render.com)
2. 新建 Web Service，连接仓库或上传代码
3. Build Command：`pip install -r backend/requirements.txt`
4. Start Command：`cd backend && uvicorn main:app --host 0.0.0.0 --port $PORT`
5. 添加环境变量后部署，记下域名

### 方案 C：自有服务器 / VPS

```bash
# 1. 上传 chatbot 文件夹到服务器
# 2. 安装依赖
cd chatbot/backend
pip install -r requirements.txt

# 3. 使用 systemd 或 supervisor 常驻运行，例如：
uvicorn main:app --host 0.0.0.0 --port 8000

# 4. 配置 Nginx 反向代理，并启用 HTTPS（Let's Encrypt）
```

### 方案 D：Docker

```bash
cd chatbot
docker build -t anyway-chatbot .
docker run -d -p 8000:8000 \
  -e OPENAI_API_KEY=你的密钥 \
  -e OPENAI_BASE_URL=你的API地址 \
  anyway-chatbot
```

## 二、在官网加入机器人脚本

拿到后端 URL 后（假设为 `https://chat.anywayflooring.com`），在官网每个页面的 `</body>` 前加入：

```html
<!-- Anyway Flooring Chatbot -->
<script src="https://chat.anywayflooring.com/widget/chatbot-widget.js" 
        data-api-url="https://chat.anywayflooring.com" defer></script>
```

把 `https://chat.anywayflooring.com` 替换成你实际的后端地址。

### 方式 1：官网和本项目同源（html 文件夹）

如果线上官网就是本项目里的 `html` 文件夹，可以用脚本批量注入：

```bash
cd chatbot/backend
python scripts/integrate_widget.py --api-url https://你的后端地址.com
```

然后重新部署 `html` 文件夹到线上服务器。

### 方式 2：官网使用 CMS / 建站后台

在后台找到「全局页脚」「自定义代码」「head/body 脚本」等位置，把上面的 `<script>` 标签粘贴进去。

### 方式 3：官网使用 Nginx / Apache 静态文件

直接编辑每个 HTML 文件，在 `</body>` 前加入上述脚本。若文件很多，可写脚本批量替换。

## 三、检查清单

| 项目 | 说明 |
|------|------|
| 后端已部署 | 能访问 `https://你的域名/widget/chatbot-widget.js` |
| 环境变量 | `OPENAI_API_KEY` 等已正确配置 |
| CORS | 后端默认允许 `*`，一般无需配置 |
| 官网脚本 | 每个页面都加了 `<script>` 标签 |
| 数据与知识库 | 已执行 `extract_products.py` 和 `build_vectordb.py` |

## 四、管理后台（可选）

如需在线上使用管理后台，需要单独部署：

1. **后端**：`chatbot/admin/admin-backend`（Node.js）
2. **前端**：先 `cd admin-frontend && npm run build`，再部署 `dist` 或由后端托管

管理后台可配置 System Prompt、快捷回复等，配置会写入 `admin-data/config.json`，供 Python 后端读取。

---

## 五、快速测试

部署完成后，在浏览器访问官网，右下角应出现聊天按钮。点击后能正常对话即表示接入成功。

## 常见问题

**Q：机器人不显示？**  
- 检查浏览器控制台是否有报错  
- 确认脚本 URL 是否正确、是否可访问  

**Q：能打开但无法回复？**  
- 检查后端日志  
- 确认 `OPENAI_API_KEY` 等环境变量是否正确  

**Q：官网和后台不同域名会有跨域问题吗？**  
- 后端已配置 CORS 允许任意来源，一般无需额外设置
